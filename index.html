UCC - Ingeniería de Sistemas

Ley Especial de Ciberdelitos

Art. 4

Acceso Indebido

Ingreso a un sistema informático sin autorización o excediendo la misma.

Art. 9

Daño de Datos

Borrado, deterioro, alteración o supresión de datos informáticos.

Art. 12

Fraude Informático

Alteración o manipulación para obtener un beneficio económico indebido.

Art. 5

Intervención de Datos

Acceso no autorizado a programas o datos contenidos en un sistema.

Art. 13

Espionaje

Obtención, revelación o difusión de datos reservados o sensibles.

Rol del Auditor

El auditor debe saber tipificar el hallazgo para derivarlo legalmente.

Caso: Incidente Universidad Alpha

Escenario: Auditoría interna detecta múltiples irregularidades en el semestre:

Evento A Estudiante modifica notas usando clave de docente.
Evento B Empleado extrae base de datos para venderla.
Evento C Alteración de pagos financieros (Fraude).
Evento D Acceso a expedientes médicos (Privacidad).

Actividad de Análisis (1h)

  1. Identificar: Qué acciones son delitos.
  2. Tipificar: Relacionar con Artículos de Ley 1042.
  3. Evaluar: Fallas de control (Contraseñas, Logins).
  4. Evidencia: ¿Qué buscaría un auditor?

El Rastro Digital (Evidencia)

Logs de Sistema

Registros de intentos fallidos, horas de acceso y direcciones IP sospechosas.

Logs de Base de Datos

Historial de transacciones (Trigger logs) que muestran el valor anterior vs. nuevo.
